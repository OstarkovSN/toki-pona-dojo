# Frontend

React 19 + TypeScript + Vite + TailwindCSS v4 + shadcn/ui. Package manager: **bun**.

## Commands

```bash
bun install              # Install deps
bun run dev              # Vite dev server ‚Üí http://localhost:5173
bun run build            # tsc + vite build (also regenerates routeTree.gen.ts)
bun run lint             # Biome check --write (auto-fixes)
bun run test             # Playwright E2E tests (headless)
bun run test:ui          # Playwright with interactive UI
bun run generate-client  # Regenerate API client from backend OpenAPI schema
bun run preview          # Preview production build locally
```

All commands also work from the repo root via workspace forwarding (`bun run dev` ‚Üí `bun run --filter frontend dev`).

## Key Files

- `src/hooks/useAuth.ts` ‚Äî Auth state. Checks `localStorage` for `access_token`. `isLoggedIn()` is a plain function (not reactive) that checks `localStorage` for `access_token`. Exported separately and used in route `beforeLoad` guards.
- `src/client/` ‚Äî Auto-generated from backend OpenAPI. Regenerate with `bun run generate-client` after any backend API change.
- `src/routeTree.gen.ts` ‚Äî Auto-generated on build by `@tanstack/router-plugin`. Do not edit. Regenerates on `bun run build` or when Vite hot-reloads a route file.
- `openapi-ts.config.ts` ‚Äî Config for client generation. Points to `./openapi.json` (local file, generated by `scripts/generate-client.sh`).
- `biome.json` ‚Äî Linting/formatting config (Biome, not ESLint/Prettier).
- `playwright.config.ts` ‚Äî E2E test config. Tests in `tests/`.

## TanStack Router Conventions

- Routes are file-based. New routes go in `src/routes/`.
- `_layout.tsx` is the authenticated layout wrapper. `_layout/` contains child routes requiring auth.
- `routeTree.gen.ts` is regenerated automatically by the Vite plugin when you add/rename route files.
- Route `head()` export sets page `<title>` ‚Äî follow the pattern `"PageName - toki pona dojo"`.

## TanStack Query

Data fetching uses TanStack Query (`@tanstack/react-query`). Use generated client services as query functions:
```ts
const { data } = useQuery({ queryKey: ['users'], queryFn: () => UsersService.readUsers() })
```

## Theme (Phase 5+)

- Tailwind v4 with CSS variables for colors. Theme defined in `src/index.css` using `@theme inline` block.
- Earth-tone/zen palette: warm parchment backgrounds, teal primary, coral destructive.
- Fonts: Lora (serif, headings) + DM Mono (code). Loaded via `<link>` in `index.html`.
- Dark mode via `next-themes` + `class` strategy. `.dark` class on `<html>`.

## Non-obvious Patterns

- **Never edit `src/client/`** ‚Äî fully regenerated. Put API call wrappers in hooks or components.
- **Never edit `routeTree.gen.ts`** ‚Äî regenerated on build or hot-reload.
- **`bun` not `npm`** ‚Äî the lockfile is `bun.lock`. Using `npm install` will corrupt deps.
- **`biome` not ESLint** ‚Äî `bun run lint` runs `biome check --write`. Config in `biome.json`.
- **Playwright tests** require the full stack running. The `playwright.config.ts` expects `http://localhost:5173` to be live.
- **`generate-client`** reads from `frontend/openapi.json` (local file). Run `bash scripts/generate-client.sh` from repo root to regenerate JSON and client (imports backend in-process, no running server needed).
- **`@` is an alias for `src/`** ‚Äî configured in `vite.config.ts`. All internal imports use `@/components/...`, `@/hooks/...`, etc.
- **PostToolUse:Edit hook runs Biome formatter after every Edit** ‚Äî always Re-Read the file before a second Edit targeting the same region, or old_string matching will fail on reformatted content (quotes/semicolons/indent style shift).
- **Use `git rm -r` for directory deletions** ‚Äî `rm -r` only removes from filesystem. To stage deletions properly in git, use `git rm -r <path>`.
- **Grep for "Items" feature produces false positives** ‚Äî Tailwind `items-center`, shadcn `FormItem`/`SelectItem`/`DropdownMenuItem`, and `localStorage.getItem` are unrelated. Known safe false positives in `AppSidebar.tsx` (nav-item `Item` type).
